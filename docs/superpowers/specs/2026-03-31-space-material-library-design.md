# 局内素材库（Space Material Library）- 设计稿

日期：2026-03-31  
参考：`J:\code\tuan-chat\material_package_PRD.md`  
实现目录：`J:\code\tuan-chat-sandbox\material-package-nav\tuan-chat-web-main`

## 1. 背景与目标

PRD 定义：
- 局外素材包（用户级）与局内素材包（space 级）数据结构一致，区别仅是作用域。
- 房间只“消费素材”：从局内素材包把素材拖进房间；素材可能包含多条消息，需按顺序批量发送。
- 支持从局外素材包（素材广场/我的素材包）导入一个完整副本到局内（space）。

本设计目标是在“局内素材库”中实现两类核心能力：
1) **发送素材**：拖拽 / 点击发送，以“消息（或消息组）”发送到指定房间。  
2) **批量与顺序**：文件夹/多选/自定义顺序，以 1/2/3 标号呈现并按队列顺序发送。  
3) **导入素材包到局内**：从素材广场或我的局外素材包导入到当前 space 的局内素材包列表（落库）。

## 2. 设计原则（收敛复杂度）

为了避免目录里出现大量“发送”按钮、视觉混乱与移动端误触：
- “目录树”只负责：**浏览、选择、拖拽、预览**
- “发送”集中在固定位置：**发送托盘/队列（Send Tray）**
- 拖拽/多选/文件夹等所有入口最终都落到同一个抽象：**发送队列**（可排序、可删除、可一次发送）

## 3. 入口与布局

### 3.1 放置位置（推荐）

放在 **Space/房间页面的侧边抽屉**中作为一个 Tab：`局内素材库`。
- 优点：符合“房间内消费素材”的心智；拖拽与发送就地完成；移动端更顺。
- 同时提供二级入口：`打开管理页`（可跳转到现有“我的素材包”式全屏管理/浮窗复用页）。

> 本次实现优先：抽屉 Tab + 托盘发送；全屏管理页可先复用现有页面作为只读/管理入口。

### 3.2 布局结构

抽屉内纵向两段：
1) **素材包目录树 + 顶部工具栏**（复用现有目录树交互：新建/导入/重命名/删除等按需求逐步接入）
2) **发送托盘/队列（固定底部）**

浮窗预览沿用现有 `MaterialPreviewFloat`（点击/拖拽打开预览）。

## 4. 数据模型与数据源

### 4.1 局内素材包数据结构

复用 `MaterialPackageContent`（`root: folder/material` 树），局内记录类型：
- `SpaceMaterialPackageRecord`（`spacePackageId`, `spaceId`, `content`, `updateTime` ...）

现有 API（已在前端封装）：
- 列表：`GET /space/materialPackage/list?spaceId=...` → `listSpaceMaterialPackages`
- 详情：`GET /space/materialPackage/{spacePackageId}` → `getSpaceMaterialPackage`
- 创建/更新/删除：`createSpaceMaterialPackage` / `updateSpaceMaterialPackage` / `deleteSpaceMaterialPackage`
- 从局外导入到局内：`POST /materialPackage/{packageId}/importToSpace` → `importMaterialPackageToSpace`

### 4.2 “素材=消息”的发送抽象

素材节点（material）包含 `messages: MaterialMessageItem[]`，每个元素可映射为一次 `ChatMessageRequest`。

**映射规则（建议）**：
- `roomId`：
  - 托盘发送：取“发送托盘目标房间”
  - 直发（拖拽/等价操作）：取 drop 目标 room（当前消息区=当前 room；房间列表项=该 room）
- `messageType/content/extra/webgal/annotations`：直接取 `MaterialMessageItem`
- `roleId/avatarId`：优先使用 `MaterialMessageItem.roleId/avatarId`，缺省则使用当前聊天输入的角色/头像

## 5. 发送托盘（Send Tray）

### 5.1 托盘包含哪些控件

固定区域包含：
- **目标房间下拉框**（默认当前 room；可切换；按 `spaceId` 记忆最近一次选择）
- 队列列表（每项左侧显示序号 1/2/3）
- `发送` 主按钮（唯一发送入口）
- `清空队列` / 单项删除（轻量）

### 5.2 目标房间选择策略

因为一个频道（space）可包含多个房间（room），选择目标房间是合理且必要的。

规则：
- 默认目标 = 当前打开的 room
- 用户手动选择一次后：记忆到 `localStorage`（key 包含 `spaceId`）
- 目标房间失效（被删/无权限）时：自动回退到当前 room，并 toast 提示一次

### 5.3 队列的发送语义

**统一规则（写死，避免歧义）**：
- 队列内部只存 **`material` 项**（每项内部可包含 1..N 条 message）。
- `folder/package` **不会作为队列项存在**：在 `enqueue()` 时立即按规则展开为 material 列表并入队。

发送模式（阶段 1）：
- **逐条发送（separate）**：按队列顺序、按 material 内部 messages 顺序，逐条调用 `sendMessageWithInsert`（用于保证顺序与逐条进度可见）。
  - 若运行环境没有 `sendMessageWithInsert`，才降级为 `batchSendMessagesAsync(requests)`，并要求后端/实现保证按 requests 顺序落库展示。

失败策略：
- 发送中展示进度与状态（见 10. 错误与边界）
- 中途失败：停止后续发送，提示“发送失败（队列第 i 项 / 该项第 j 条）”，队列保留，用户可重试

## 6. 目录树交互（选择/预览/入队）

### 6.1 点击与预览

- 单击素材（material）：打开浮窗预览（不直接发送）
- 双击名称：重命名（复用你已有的“原地重命名”模式）

### 6.2 入队规则（统一）

所有入口最终都调用同一函数：`enqueue(items, mode)`。

入口：
- 拖拽素材/文件夹到托盘：入队
- 多选后点击“加入队列”：入队
-（可选）工具栏提供 `+队列`：把当前选中入队

**文件夹展开顺序（目录顺序）**：
- 深度优先（DFS），按 `children` 当前顺序遍历
- 只把 `material` 叶子作为发送项入队（folder 本身不发消息）

## 7. 多选与顺序（带 1/2/3 标号）

目标：支持用户决定发送顺序，同时不在目录树里放太多按钮。

阶段 1（最小可用，稳态方案）：
- 支持多选（Ctrl/⌘ 单选叠加；Shift 范围）
- **树上不显示 1/2/3 序号**（避免“点击顺序”状态机复杂、易出不可预测行为）
- 入队顺序：
  - 若存在 Shift 连续范围：按目录顺序入队
  - 其它情况：按目录顺序入队（稳定可预期）
- 顺序调整全部在托盘队列完成（拖拽重排），托盘中显示 1/2/3…

阶段 2（增强，可选）：
- 树上显示 1/2/3 序号（按“选择序列”而非目录序）
- 支持“按点击顺序入队”（需要完整定义：取消选择是否重排、跨文件夹显示、折叠/搜索后的序号保持等）
-（可选）同文件夹内拖拽改变 `children` 顺序（影响“目录顺序发送”）

## 8. 拖拽到房间（直发）

除了托盘入队，还支持更直接的“拖到房间发送”。**阶段划分如下**：
- 阶段 1：必须支持“拖到当前 room 消息区直发”（对齐 PRD 的“拖进房间”心智）
- 阶段 2：支持“拖到房间列表某个 room 直发”（桌面端能力；移动端一般不可用）

直发拖拽能力：
- 从目录树拖拽 `material/folder` 到左侧房间列表某个 room：
  - `material`：按其 message(s) 逐条发到该 room
  - `folder`：按目录顺序展开后逐条发到该 room
- 从目录树拖到聊天消息区（当前 room）：等同“发到当前 room”

实现策略：
- 沿用现有 DnD payload（当前外层目录使用 `MaterialPreviewPayload`），扩展/复用到“局内素材库”场景，保证 drop handler 能解析出“目标节点”和“所属 spacePackageId/path”。

### 移动端等价路径（阶段 1 必须提供）

移动端通常不具备可靠的拖拽手势，因此阶段 1 在移动端以“等价操作”满足“直发到当前房间”的需求：
- 在浮窗预览（`MaterialPreviewFloat`）或素材长按菜单提供 `发送到当前房间` 动作
- 该动作语义与“拖到当前消息区直发”等价：按顺序逐条发送 material（或 folder 展开后的 materials）

## 9. 从局外导入到局内（落库）

入口：局内素材库顶部工具栏的 `导入素材包`。

流程：
1) 弹窗选择来源：`素材广场` / `我的素材包`
2) 选择一个素材包后点击导入
3) 调用 `importMaterialPackageToSpace(packageId, { spaceId })`
4) 刷新 `listSpaceMaterialPackages(spaceId)`，并在目录树中 reveal 新包

约束：
- 导入到“当前 space”，不在此流程中选择 space（避免多一步）
- 导入后是副本：后续编辑不影响局外原包

## 10. 错误与边界

- 未选择目标房间：托盘发送按钮禁用，并提示“请选择房间”
- 队列为空：发送按钮禁用
- 没有局内素材包：展示空态 + 引导“导入素材包”
- 发送失败：停止并保留队列，允许重试（从失败处继续 / 重试失败项 / 全部重试）
- 发送状态（托盘队列项）：
  - `pending` / `sending` / `sent` / `failed`
  - material 内部 messages 也维护同样的状态（内部状态，不一定全部展示，但用于 i/j 定位与续发）
  - 发送中禁用重排与清空；提供“停止发送”按钮
- material 多消息的失败定位：提示“队列第 i 项 / 该项第 j 条”
- 目标房间权限不足/不可写：发送前置校验（下拉列表过滤不可写房间），若运行时失败则归类为 `failed`
- 移动端：默认以“点选/加入队列/发送”为主；拖拽可不支持或弱化（给出替代操作路径）

### 停止 / 重试的确定语义（阶段 1 写死）

- 停止发送：不会取消已发出的网络请求；在“当前正在发送的那一条 message”结束后停止后续发送。
- 从失败处继续：从“队列第 i 项 / 该项第 j 条”开始继续发送；已 `sent` 的不会重复发送。
- 重试失败项：从该项第 1 条开始整项重发（可选）或从失败 j 开始续发（推荐与“从失败处继续”一致）。

### 可访问性（A11y）约束（阶段 1 必须满足）

- 托盘“目标房间下拉/发送/清空/停止发送”提供明确 `aria-label`
- 队列序号与状态需要可读文本（例如“第 3 项：失败”）
- 队列顺序调整必须提供非拖拽路径：
  - 每项提供“上移/下移”按钮（或快捷键 Alt+↑/↓），读屏可感知顺序变化
- 键盘操作：
  - Tab 可聚焦到目录树可操作项、托盘下拉与按钮
  - Enter/Space 触发“加入队列/发送”等按钮
- toast 文案应足够描述失败原因与位置（第 i 项/第 j 条）

## 11. 验收标准（阶段 1）

- 在 space 页面打开“局内素材库”Tab，可看到局内素材包目录树与浮窗预览
- 托盘可选择目标房间，能把素材/文件夹入队，并显示 1/2/3 顺序
- 点击“发送”能按队列顺序逐条发送（material 多消息按 messages 顺序）
- 桌面端：支持把 `material/folder` 拖到“当前 room 消息区”直发（按顺序逐条发送）
- 移动端：在浮窗/长按菜单提供“发送到当前房间”，语义与直发一致
- 支持从“素材广场/我的素材包”导入素材包到当前 space（落库）并刷新目录树
